# Техническое задание и Логика работы приложения (Specs)

## 0. Аутентификация и Безопасность
Полноценная система входа и защиты данных.
- **Экран входа:** Минималистичный интерфейс (Логин, Пароль). Валидация полей на лету.
- **Механизм:** JWT (JSON Web Tokens). Используется пара Access + Refresh токенов.
- **Сессия:**
    - При старте приложения проверяется наличие сохраненных токенов.
    - Если токены есть -> сразу на Главный экран.
    - Если токенов нет или они истекли (и Refresh не помог) -> Экран входа.
- **Выход:** Кнопка "Выйти" в AppBar на экране списка проектов. Удаляет токены и перебрасывает на экран входа.

## 1. Главный экран
- Приветствие, дата/время.
- Карточка "Создать новый объект".
- Список "Недавние объекты".
- Быстрый поиск (адрес, телефон, дата).

## 2. Раздел "Объекты"
Система фильтров: новизна, алфавит, дата обновления, тип объекта.

### 2.1 Карточка объекта в списке
Компактное отображение информации о проекте в списке объектов:

- **Адрес и Домофон:**
    - Адрес: Крупный шрифт (16px, Bold, Black87), всегда на первом месте.
    - Домофон: Мелкий текст (11px, Grey 500), показывается под адресом с префиксом «домофон: ...»
    - **Центрирование:** Если код домофона отсутствует, адрес центрируется по вертикали внутри блока (между верхней границей и следующим контентом).
- **Даты:**
    - «Создан: DD.MM.YYYY» — всегда показывается.
    - «Изменен: DD.MM.YYYY» — **показывается только если прошло ≥ 2 часа** после создания объекта.
- **Иконки типов объектов:**
    - Новостройка: `Icons.apartment`
    - Вторичка: `Icons.home`
    - Коттедж: `Icons.villa`
    - Офис: `Icons.business`
    - Другое: `Icons.category` (отличается от Офиса)
- **Интерактивность:** Hover-эффект, курсор pointer, навигация в детальный экран по клику.

### 2.2 Поиск по объектам
Поиск реализован как **Overlay-панель** (Layered Architecture):

- **Взаимодействие:**
    - Иконка поиска в AppBar (всегда видима).
    - При нажатии сверху выезжает панель поиска (`SlideTransition`), перекрывая верхнюю часть списка.
    - Фон затемняется прозрачным барьером.
    - Клик по барьеру или крестику закрывает поиск.
- **Логика:** Фильтрация списка «на лету» (по вхождению строки в адрес). Панель не смещает контент, а накладывается поверх (Stack).

### 2.3 Фильтры объектов
Система фильтрации доступна через иконку фильтра в AppBar (Badge при активных фильтрах).

- **Сортировка:**
    - «Сначала новые» (по дате создания, DESC)
    - «Сначала старые» (по дате создания, ASC)
    - **«Наиболее $»** — сортировка по сумме работ (прибыль DESC)
    - **«Наименее $»** — сортировка по сумме работ (прибыль ASC)
- **Источник:** Фильтрация по источнику лида (Владимир, Другое).
- **Тип объекта:** Новостройка, Вторичка, Коттедж, Офис, Другое.
- **UI:** `ChoiceChip` с `materialTapTargetSize: shrinkWrap` (без splash-эффекта).

### 2.4 Layout и Отступы
- **Специальная область (Safe Area):** Список объектов имеет нижний отступ `100px` (`ListView(padding: const EdgeInsets.only(bottom: 100))`.
- **Цель:** Исключить перекрытие нижних карточек объектов кнопками управления (Add, Search) при скролле.

- **Премиальный Хедер "Об объекте" (v6):**
    - Расположен в верхней части вкладки «Этапы».
    - **Заголовок:** «Об объекте» (ранее «Объект»).
    - **Инфо-блоки:** 2 элемента (Заказчик, Источник) расположены **вертикально** с цветными иконками.

    - **Стиль:** Белый фон, тень `blur: 15`, вертикальная полоса Индиго (6px) слева.
- **Интерфейс:** TabBar (Этапы | Щиты | Файлы). 
- **Данные:** Заказчик (client_info), Тип объекта, Источник (source).
- **Файлы:** PDF проекта, фото, схемы. 
    - **Интерфейс:** Компактная сетка (6 колонок), мини-карточки (v4 Premium Style).
    - **Спойлеры:** Категории с **> 5 файлами** сворачиваются по умолчанию. Категории с **≤ 5 файлами** раскрыты.

    - **Лимиты загрузки:**
        - **Количество:** Максимум **12 файлов** на весь проект (суммарно по всем категориям).
        - **Размер:** Максимум **20 МБ** на один файл.
        - **Типы:** `jpg, jpeg, png, webp, heic, heif, pdf, docx, xls, xlsx, txt, zip, mp4, mov`.
    - **Функции:**
        - **Переименование:** Безопасное переименование (расширение сохраняется).
        - **Скачивание:** "Сохранить как..." в выбранную папку.
        - **Cleanup:** Временные файлы (для просмотра/шаринга) удаляются при закрытии приложения.
- **Интерфейс:** TabBar (Этапы | Щиты). 
- **Вкладка "Щиты":** Единая инженерная карта, содержащая список всех щитов объекта.
- **Цветовая схема:** Унифицирована на **Indigo** для всех типов щитов и диалогов.
- **Порядок типов:** Силовые -> Слаботочка -> LED.
- **Интерфейс:** Карточки с акцентной полосой (Blue/Amber/Purple).
- **Типографика заголовков (v5):** Стиль "Label-Value". Тип щита (мелкий капс) над названием (крупный жирный шрифт с letterSpacing -0.3).

## 3. Раздел "Просчеты" (Логика смет) - В РАЗРАБОТКЕ
### Функции:
- Создать предв. смету по шаблону24. - **Авторасчет:** Генерация сметы работ на основе материалов. Поддерживается:
    - **Агрегация:** Группировка разных материалов в одну работу (через `aggregation_key`).
    - **Связь 1-к-1:** Прямая связь через `related_work_item`.
- **Интеграция:** Перенос данных из "Инженерной карты" (Щиты) напрямую в смету.
    - **Fallback Logic:** Если товар не найден по ключу (`mapping_key`), создается позиция-заглушка "ВНИМАНИЕ: Не найден...", цена 0.
    - **Replacement Logic:** При повторном импорте существующие позиции удаляются и создаются заново. Для корпусов (щитов) действует правило: если в проекте есть хоть один щит, все старые корпуса в смете материалов удаляются перед импортом.

### 3.1 Логика добавления позиции (Add Item Flow):
1. Нажатие **"+"** -> Открывается **Search Dialog** (Поиск по каталогу).
2. Выбор элемента -> Открывается **QuantityInputDialog**:
   - **Smart Calculator:** Ввод `Общий объем` / `Мы` / `Контрагент` (авторасчет третьего).
   - **Валюта:** Тумблер выбора валюты (USD/BYN) для данной позиции.
   - **Цена:** Возможность изменить базовую цену сразу при добавлении.
3. Нажатие **"Добавить"** -> Позиция попадает в смету с выбранными параметрами.

## 4. Раздел "Финансовый монитор"
Экран для отслеживания финансового состояния проектов и этапов.

### 4.1. Основные функции
1. **Список проектов с неоплаченными этапами:**
   - Отображение адреса проекта, общей суммы и количества этапов.
   - Раскрывающийся список этапов (expandable).
   - Pull-to-refresh для обновления данных.

2. **Карточка проекта:**
   - **Плашка этапов:** Иконка слоев + количество этапов, градиентный фон (зеленый).
   - **Сумма:** Отображение общей суммы проекта в USD/BYN.
   - **Раскрытие:** Клик по карточке раскрывает/сворачивает список этапов. При раскрытии вокруг всей карточки появляется **зеленая рамка** (width 2.0) для визуальной изоляции активного проекта.

   - **Карточка этапа:**
     - **Название:** Отображаемое имя этапа (например, "Этап 1 (Черновой)").
     - **Дата изменения:** Динамический формат ("Сегодня", "Вчера", "3 дн. назад", "07.02.2026").
     - **Цвет сумм:** Основные суммы (USD/BYN) отображаются **черным цветом**.
      - **Кнопка подтверждения:** Стильная кнопка `OutlinedButton.icon` с ярко-зеленой обводкой и иконкой галочки.
      - **Выравнивание:** Все элементы строки этапа строго **центрированы по вертикали**.
      - **Интерактивность:** Hover-эффект (0.08 opacity) для всех кнопок действий.

4. **Глобальные настройки:**
    - **Смета контрагента:** Текстовое поле для заметок о смете контрагента.
    - **Финансовые заметки:** Общие финансовые заметки.
    - Оба поля адаптивные (растут по содержимому, `minLines: 2`), имеют внутренние отступы **14/10px** и внешние **16px**.

### 4.2. Логика отображения сумм
- **Наша часть:** `our_amount_usd` / `our_amount_byn` из API.
- **Общая сумма:** `our_amount + external_amount` (контрагент).
- **Формат:** "200$" (наша) + "из 350$" (общая).
- **Условие:** Дополнительная строка "из XXX$" показывается только если `external_amount > 0`.

### 4.3. Обновление данных
- **Автоматическое:** При возврате на экран (Riverpod механизм).
- **Ручное:** Жест pull-to-refresh (потянуть вниз).
- **Отмена Timer:** Удален периодический автообновление (был каждые 30 секунд).

## 4.X. Раздел "Итоговые сметы" (Legacy)
1. **Для клиента:** Полное количество (quantity), кнопка "Оплачено".
2. **Для контрагента:** Учет доли контрагента через `employer_quantity`.
3. **Валюта:** Поддержка USD/BYN для каждой строки (выбор при создании).
4. **Таблица Итого:** Отображение в виде Dashboard-сетки (Всего / Контрагент / Наши) с разбивкой по валютам (USD/BYN).
5. **Группировка:** Автоматическая группировка позиций по категориям справочника (Кабели, Автоматика и т.д.).
6. **Заметки и Примечания:**
    - **Internal Notes:** Личные заметки (не видны клиенту).
    - **Estimate Remarks:** Примечания для сметы (попадают в отчет/PDF).
    - **Интерфейс:** Блок заметок на цветной подложке (как в Total), поля ввода с компактным шрифтом (12px) и ручным сохранением кнопкой `OutlinedButton.icon`. 
    - **Дефолтное значение:** Для материалов ("Не учтен вводной кабель"). Кнопка сохранения скрыта, если дефолт не менялся.
7. **Умный калькулятор (Smart Calculator):** При вводе/редактировании работ, пользователь может ввести любые 2 значения из 3-х (`Общий объем`, `Мы`, `Контрагент`), а третье рассчитается автоматически по формуле: `Общий = Мы + Контрагент`.
8. **Наценка (Markup):** Во вкладке материалов доступна наценка (markup) от 0% до 100%. Наценка увеличивает стоимость материалов для клиента, не меняя закупочную цену для нас/контрагента.
9. **Видимость цен (Client Mode):** Вкладка материалов позволяет переключать режим "С ценами / Без цен" для удобства демонстрации клиенту.
10. **AppBar Overflow Menu (Действия):** Вторичные действия перенесены в выпадающее меню ("Три точки") в AppBar:
    - **Стилизация:** Скругленные углы (16px), иконки, разделители групп.
    - **Состав:**
        - *Отображение:* "Показывать цены" (Чекед-бокс, только для вкладки Материалы).
        - *Опасная зона:* "Очистить смету" (Красный цвет, внизу списка).
        - *Инструменты:* Импорт, Шаблоны.
    - **FAB:** Единственная плавающая кнопка для **Добавления позиции**.
    - **Safe Interaction:** Меню не перекрывает контент критически.
11. **Контекстные диалоги:** Кнопки действий окрашиваются в цвет вкладки (Зеленый = Работы, Синий = Материалы) для визуальной навигации.
12. **Меню Действий (Actions Menu UI Refinement):**
    - **PDF & Text:** Две независимые кнопки-иконки в AppBar для быстрого доступа к отчетам.
    - **Вторичные действия:** Скрыты в Overflow Menu ("Три точки").
    - **Визуальные стандарты:** См. детальные спецификации (иконки, цвета, верстку предпросмотра) в [DESIGN.md](./DESIGN.md).

13. **Умные отчеты (Smart Reports V7):**
    - **Генерация:** Мгновенная генерация текста на клиенте.
    - **Материалы (Вкладки):**
        - *Материал*: Только кол-во и ед. изм (без цен).
        - *Материал с ценами*: Базовые закупочные цены.
        - *Материал с + %*: Цены с учетом наценки (Markup).
    - **Работы (Вкладки):**
        - *Наши*: Объем за вычетом доли контрагента.
        - *Контрагент*: Доля контрагента (Заголовок "Адрес... - ТВОИ").
        - *Заказчик*: Полный объем работ.
    - **Логика расчетов:**
        - **Округление:**
            - *Работы*: До целых чисел (Round).
            - *Материалы*: До 2 знаков после запятой.
            - *Общее*: Удаление всех лишних нулей (`10.00` -> `10`, `12.50` -> `12.5`).
        - **Формулы (Footer):**
            - *Твои*: Детальная формула (`10$ + 5$ = 15$; 100р`). Разделитель `+` используется, если во второй валюте только 1 позиция.
            - *Наши*: Расчет от полного объема этапа (`Всего(Этап) - Твои`). Если разница в рублях равна 0, она не выводится. Если у контрагента 0р, а у нас есть, выводится `+ Xр`.
    - **UI (Mobile-First):** Навигация через **Wrap Chips** (переносятся на новую строку), пастельные тона кнопок, цветные разделители, черная кнопка закрытия.
    - **Примечания (Remarks):** В отчеты попадают только "Примечания для сметы" (`remarks`). Внутренние "Заметки" (`notes`) остаются только в приложении.

13. **Экспорт в PDF (Native PDF Export):**
    - **Технология:** Генерация нативного PDF-документа с поддержкой кириллицы (OpenSans).
    - **Варианты:** Полное соответствие типам текстовых отчетов (Работы: Все/Твои/Наши; Материалы: Без цен/С ценами/+%).
    - **Дизайн:** Профессиональная верстка, заголовок "Электромонтаж" (с подчеркиванием во всю ширину), "ИТОГО" серым цветом, футер "Электромонтажные работы".
    - **File Opening:** Файл сохраняется во временную папку и автоматически открывается в системном приложении (PDF Viewer) без диалога печати.
    - **Sharing:** Возможна прямая отправка файла через системное меню 'Поделиться' (Share Plus).
    - **Округление:**
        - *Работы*: Цены и Суммы округляются до целых.
        - *Материалы*: Цены и Суммы до 2 знаков.


## 5. Статистика и Аналитика
- **Финансы за период:** Отображение общего дохода (USD/BYN) за выбранный период (Месяц, Год, Все время).
- **Динамика работ:** Линейный график стоимости созданных работ/материалов (по дате создания).
- **Структура:** Круговые диаграммы по Источникам и Типам объектов.
- **Чистая прибыль:** (В планах) Расчет рентабельности.

### 6. Инженерные сети (Инженерная карта / Вкладка "Щиты")
Данные фиксируются для истории и автоматизации сметы. Реализована Multi-shield архитектура.
- **Редактирование:** Название, тип монтажа, заметки.
- **Заметки (Notes):** Кнопка "Заметка" с пульсацией при наличии текста.
- **Диалоги:** Все диалоги (Добавление группы, LED зоны, Линий, Заметок, Добавление/Редактирование щита) используют **Indigo** как основной акцентный цвет.
- **Выпадающие списки:** Используют акцентный фон `shade50` (синеватый) для кнопок и самого меню (Solid Accent Style).

#### 6.1 Силовой щит (ShieldGroup):
- **Smart Logic:** Тип устройства, Номинал и Полюса.
- **Нормализация:** RegEx на бэкенде (2п -> 2P, 16а -> 16A).
- **Расчет модулей:** Автоматический modules_count.
- **Подбор щита:** Стандартный ряд [..., 12, 24, 36, 48, 60, 72, 96, 120, 144].
- **Smart Logic:**
    - **0 модулей:** Корпус не создается.
    - **> 144 модулей:** Создается предупреждение "Индивидуальный расчет".
    - **Тип монтажа:** Учитывается (Встраиваемый/Навесной) при формировании ключа поиска.
    - **Fallback:** Если корпус не найден в каталоге, создается предупреждение с указанием типа монтажа.

#### 6.2 Зоны LED (LedZone):
- **Авто-щит LED (Логика подбора):**
    - 1-2 транса -> "Переместить в слаботочный щит"
    - 3-4 транса -> "24 модуля"
    - 5-9 трансов -> "36 модулей"
    - 10-12 трансов -> "48 модулей"
    - 13-15 трансов -> "60 модулей"
    - Более 15 -> "Индивидуальный расчет"
- **Реализация:** Использует отдельные ключи каталога (`shield_media_enclosure_...`), разделяя их с силовыми щитами.

#### 6.3 Слаботочка и Интернет (Multimedia):
- **Логика подбора корпуса:**
    - до 4 линий -> "24 модуля"
    - 5-10 линий -> "36 модулей"
    - Более 10 -> "Индивидуальный расчет"
- **Реализация:** Использует те же ключи `media`, формируя отдельную позицию в смете. Учитывает тип монтажа.
- **Интерфейс:** Гибридный ввод количества линий (0-10 + ручной ввод) и стилизованные заметки.

### 6.4 Шаблоны:
- Кнопка "Заполнить по шаблону" копирует набор строк из справочника в текущий проект. **Внимание:** Перед добавлением новых строк текущий раздел (работы, материалы или наполнение щита) полностью очищается.
