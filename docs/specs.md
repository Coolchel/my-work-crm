# Техническое задание и Логика работы приложения (Specs)

## 1. Главный экран
- Приветствие, дата/время.
- Карточка "Создать новый объект".
- Список "Недавние объекты".
- Список "Недавние просчеты".
- Быстрый поиск (адрес, телефон, дата).

## 2. Раздел "Объекты"
Система фильтров: новизна, алфавит, дата обновления, тип объекта.

### Карточка объекта содержит:
- **Статус (Workflow):**
    - Новый (только адрес/телефон).
    - Предпросчет (есть смета мат. и работ 1-2 этап).
    - Этап 1 завершен (итоговая смета этап 1).
    - Этап 2 завершен (итоговая смета этап 2 + материал этап 3).
    - Этап 3 завершен.
    - Доп. этапы (под-этапы).
- **Данные:** Адрес, Домофон, Описание, Заказчик, Тип объекта (Новостройка/Вторичка/Коттедж...), Источник заказа.
- **История:** Дата создания и лог изменений (автоматически).
- **Файлы:** PDF проекта, фото до/после, схемы щитов (отдельный тип файла).
- **Сметы:** Сохраненные сметы материалов и работ.

## 3. Раздел "Просчеты" (Логика смет)
### Функции:
- Создать предв. смету (материалы/работа) по шаблону.
- Редактирование смет (добавление/удаление позиций).
- **Авторасчет:** Генерация сметы работ на основе сметы материалов.

### Типовые шаблоны смет:
- **Материалы 1-2 этап:** ~30 позиций (Кабели, подрозетники, автоматика, расходники).
- **Работа 1-2 этап:** ~20 позиций.
- **Материалы 3 этап:** ~15 позиций (Розетки, выключатели, рамки).
- **Работа 3 этап:** ~10 позиций.

### Алгоритм авторасчета (Связь Материалы -> Работа):
*Реализуется на Бэкенде (Python).*
- Работа просчитывается на основании количества материалов и их категорий.
- Пример: Если в смете материалов "Кабель" (сумма всех типов по категории) = 500м -> В смету работ добавляется "Монтаж кабеля" = 500м * Коэффициент.
- Пример: Кол-во подрозетников -> Кол-во высверливаний + вмазки.

## 4. Раздел "Итоговые сметы" и Финансы
Делится на 2 части:
1.  **Для клиента (Чистовая):** Упрощенный вид (Название, Дата, Сумма).
    - Отображается полное количество работ/материалов (поле `quantity`).
    - Кнопка "Оплачено" (архивация сметы).
    - Сумма неоплаченного долга.
2.  **Для подрядчика/внутреннее (Кухня):**
    - **Учет субподряда:** Реализован через поле `contractor_quantity` в каждой позиции сметы.
    - *Логика:* Если `quantity`=100, а `contractor_quantity`=80:
        - Клиенту считаем: 100 * цена.
        - Расход на подрядчика: 80 * цена.
        - Наша прибыль: (100 - 80) * цена.
    - **Валюта:** Возможность указывать валюту (USD/BYN) для каждой строки (актуально для докупок материалов).
    - Доп. заметки по финансам подрядчика (авансы, долги).

## 5. Статистика
- Кол-во просчитанных смет за месяц.
- Кол-во новых объектов.
- Финансы (Выручка общая / Расходы на подрядчиков / Чистая прибыль).

## 6. Права доступа
- Сейчас: Admin (полный доступ).

## 7. Инженерные сети (Щиты и LED)
Отдельный модуль для фиксации конфигурации оборудования (чтобы не забыть через месяц). Данные не влияют на финансовую смету напрямую, это "Инженерная карта".

### Сущности:
1.  **Группы щита (`ShieldGroup`):**
    - Поля: Устройство (напр. "Диф 16А"), Зона (напр. "Посудомойка").
2.  **Зоны подсветки (`LedZone`):**
    - Поля: Трансформатор (напр. "Блок 100Вт"), Зона (напр. "Ниша шторы").

### Шаблоны и Автоматизация:
- В системе хранятся шаблоны щитов (напр. "Типовая двушка") и шаблоны LED.
- **Функция:** В приложении кнопка "Заполнить по шаблону".
- **Логика:** Сервер копирует строки из шаблона в текущий проект. Далее пользователь может их редактировать/удалять вручную.
