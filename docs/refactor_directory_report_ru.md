# Краткий отчет по рефакторингу "Справочник"

## Что сделано
1. Добавлена инфраструктура универсального справочника на backend:
   - разделы (`DirectorySection`),
   - позиции разделов (`DirectoryEntry`),
   - уникальность кода позиции в рамках раздела.
2. Добавлен CRUD API для `directory-sections` и `directory-entries`.
3. Добавлен endpoint `POST /api/directory-sections/bootstrap/`, который синхронизирует системные справочники из текущих `choices` моделей (статусы проекта, типы объектов, этапы, валюты, типы щитов, типы устройств и т.д.).
4. На frontend переписан экран "Справочник":
   - вкладка "Системные разделы" с просмотром и CRUD позиций,
   - вкладка "Каталог" с CRUD категорий и CRUD позиций каталога,
   - компактные строки списков,
   - диалоги с шапкой и скруглениями в стиле экранов приложения.
5. Добавлен новый frontend-репозиторий `DirectoryRepository` для единого доступа к API раздела.

## Самопроверка
- Проверил backend-проверки Django (`manage.py check`) — ошибок нет.
- Проверил, что миграция создаёт только новые таблицы справочника и не вносит лишних изменений.
- Проверил, что в настройках переход открывает обновлённый экран справочника (через существующий `CategoryListScreen`).

## Ограничения окружения
- В контейнере отсутствует `dart`, поэтому автоформатирование и запуск `flutter analyze/test` выполнить не удалось.
- Скриншот интерфейса не делал, т.к. веб/desktop приложение не запускалось в текущем окружении (нет готового поднятого сервиса для browser tool).
