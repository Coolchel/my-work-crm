# Руководство по Дизайну (UI/UX Design Guide)

Этот документ описывает визуальные стандарты и принципы взаимодействия (UX) приложения Smart Electric CRM. Все новые компоненты должны соответствовать данным правилам для поддержания «премиального» вида и удобства работы экспертов.

---

## 1. Философия и Эстетика
- **Стиль:** Material 3 с элементами Glassmorphism и «Solid & Clean» дизайна.
- **Ощущение:** Профессиональный инструмент, который выглядит «дорого».
- **Принципы:** Скорость ввода, визуальная иерархия, мгновенная обратная связь.

| Цвет | Использование | Тональность |
| :--- | :--- | :--- |
| **Indigo** | Основной цвет (Primary), заголовки этапов, кнопки PDF. | Акцентный, глубокий. |
| **Teal** | Слаботочные щиты (header), Текстовые отчеты. | Спокойный, технологичный. |
| **Green** | Работы, итоговые суммы "Наши". | Положительный, финансовый успех. |
| **Blue** | Материалы, системные уведомления. | Информационный, чистый. |
| **Orange** | Силовые щиты (header), Наценка, критические акценты. | Предупреждающий, энергичный. |
| **Purple** | LED щиты (header). | Современный, ассоциируется с RGB. |
| **Red** | Деструктивные действия (удаление). | Требует осторожности. |
| **Dark Grey** | Контент, вторичные действия (`0xFF374151`). | Нейтральный, фокусирующий. |

## 3. Стандарты Диалоговых Окон (Premium Dialogs)

Все диалоги в приложении должны следовать единой структуре «Premium Dialog».

### 3.1. Анатомия диалога
- **Контейнер (`buildPremiumContainer`):**
    - Скругление углов: **24px**.
    - Максимальная ширина: обычно **400px** (или 720px для предпросмотра).
    - Тень (`BoxShadow`): Цвет темы с прозрачностью **0.15**, размытие (blur) **20px**, смещение по Y **10px**.
- **Шапка (`buildPremiumHeader`):**
    - Фон: Цвет темы с прозрачностью **0.12**.
    - Состав: Иконка (слева), Центрированный заголовок (Bold, 18px), Кнопка закрытия (справа).
    - Скругление верхних углов: **24px**.

### 3.2. Поля ввода
- **Стиль:** Чистые поля без заливки фона.
- **Метки:** Обязательное использование плавающих меток (`floating labels`).
- **Скругление:** Границы полей скруглены на **12px**.

- **Скругление:** Границы полей скруглены на **12px**.

### 3.3. Премиальные Выпадающие Списки (Custom Dropdowns)
Стандартные виджеты `DropdownButton` и `PopupMenuButton` имеют визуальные артефакты (отступы, разделители, серый фон M3). Для обеспечения премиального вида используется паттерн **Custom Trigger + Metadata Menu**.

- **Триггер (Input Field):**
    - Реализация: `InkWell` внутри `Container` с декорацией.
    - Курсор: Строго `SystemMouseCursors.click` (Hand cursor).
    - Визуализация: Иконка стрелки (`arrow_drop_down`) всегда справа.
- **Меню (Popup):**
    - Вызов: Метод `showMenu` с точным позиционированием (`LayoutBuilder`).
    - Стиль:
        - `surfaceTintColor: Colors.transparent` (отключение тонирования M3).
        - `padding: EdgeInsets.zero` (у элементов меню).
        - Элементы меню обернуты в `Container` для отступов.
    - Контент:
        - Иконки: Цветные иконки для визуальной дифференциации (Flash для тока, Bolt для полюсов).
        - Разделители: `Divider` не используется, границы задаются стилем контейнера.

### 4.1. Вкладки и Списки
- **TabBar:** Использует первичные цвета вкладок (Зеленый для Работ, Синий для Материалов).
- **Карточки позиций:**
    - Группировка по категориям каталога.
    - Компактный вид с четким разделением объемов (Всего / Мы / Контрагент).

### 4.2. Кнопки AppBar (Export Actions)
- **Стиль:** Минимализм, только иконка.
- **Размер иконок:** Крупные (28px).
- **Tooltip:** Обязательное описание действия ("PDF смета", "Текстовая смета").
- **Иконки:**
    - PDF: `Icons.auto_graph_rounded` (Deep Purple).
    - Text: `Icons.segment_rounded` (Orange/Teal).

### 4.3. Меню действий (FAB / Speed Dial)
- **Backdrop:** При открытии FAB экран затемняется, но сохраняет возможность прокрутки.
- **Цвета кнопок:** Соответствуют активной вкладке (Зеленые в работах, Синие в материалах).

## 5. Окно предварительного просмотра (Report Preview)
- **Размер:** Фиксированная высота **760px** для предотвращения «прыжков» интерфейса.
- **Иконка:** `Icons.article_rounded`.
- **Переключатели (Chips):** Стиль `ChoiceChip` без рамок, с цветным фоном (shade100/200).
- **Кнопка действия:** Центрированная кнопка "Копировать" шириной **220px** с иконкой `Icons.copy_rounded`.

## 6. Интерактивность и Обратная связь
- **Курсор:** Всегда использовать `SystemMouseCursors.click` для кнопок, чипов и интерактивных карточек.
- **Hover-эффект:** Использование прозрачного наложения цвета фона (`fg.withOpacity(0.1)`) при наведении.
- **Подтверждения:** Любое массовое или деструктивное действие (удаление всего, применение шаблона) требует вызова `ConfirmationDialog`.
- **Навигация (Pattern):** После успешного сохранения данных (например, при создании шаблона) диалог должен закрываться, возвращая пользователя на экран, с которого был вызван, без лишних промежуточных окон и без перезагрузки всего дерева (устранен «двойной pop»).

## 8. Инженерный раздел (Shield Management)

### 8.1. Визуальная иерархия щитов
### 8.1. Визуальная иерархия щитов
- **Шапка (Header):** Использует цветовой код для быстрой идентификации типа щита:
    - **Orange (Оранжевый):** Силовой щит (`Colors.orange.shade800`).
    - **Purple (Фиолетовый):** LED щит (`Colors.purple.shade600`).
    - **Teal (Бирюзовый):** Слаботочный щит (`Colors.teal.shade600`).
- **Топ-блок (Top Row):** Разделен на две зоны:
    - **Слева (Stats):**
        - Заголовок конфигурации (например, "УСТРОЙСТВА ЩИТА:") — шрифт Heavy, цвет `Color(0xFF374151)`.
        - Статистика одной строкой через запятую (например, "5 позиций, 12 модулей").
    - **Справа (Mounting):**
        - Заголовок "МОНТАЖ:" (черный цвет, без иконки) расположен *слева* от переключателя.
        - Переключатель типа монтажа (Внутренний/Наружный).
- **Зонирование:** Карточка щита разделена на блок настроек и блок содержимого. Содержимое обернуто в декоративный контейнер.

### 8.2. Концепция Нейтральности и Акцентов
- **Основные действия (Template, Edit, Delete):**
    - Расположены внутри карточки, в правом нижнем углу топ-блока.
    - Выполнены в **нейтральном стиле** (`OutlinedButton`, серый цвет).
    - Порядок: **Шаблон -> Редактировать -> Удалить**.
    - Иконки: `copy_all_rounded`, `edit_outlined`, `close_rounded`.
- **Кнопка «ДОБАВИТЬ»:**
    - Расположена в **заголовке списка** внутри контента щита.
    - Выровнена по правому краю.
    - Стиль: `OutlinedButton`, серый цвет, иконка `add_rounded`.
- **Действия с шаблонами:** Диалоги выбора и сохранения шаблонов наследуют акцентный цвет щита (Orange/Purple) для заголовков и иконок.

### 8.3. Группировка
- Использование тонких разделителей и отступов (8px) для сохранения компактности.
- Заголовки групп и зон — темно-серый `Color(0xFF374151)` для читаемости на белом фоне.

### 8.4. Цветовое кодирование устройств
Каждый тип устройства в силовых щитах имеет уникальный цвет для мгновенной визуальной идентификации:

| Тип устройства | Цвет | Семантика |
|----------------|------|-----------|
| Рубильники (`load_switch`) | Red.shade600 | Критический элемент, главный выключатель |
| УЗО (`rcd`) | Amber.shade700 | Предупреждение, защита от утечки |
| Автоматы (`circuit_breaker`) | Blue.shade600 | Стандартная защита, стабильность |
| Диф. автоматы (`diff_breaker`) | Purple.shade600 | Премиум защита, комбинированная |
| Реле (`relay`) | Teal.shade600 | Автоматизация, технологии |
| Контакторы (`contactor`) | Orange.shade700 | Силовая коммутация, высокая мощность |
| Другое (`other`) | BlueGrey.shade600 | Нейтральность |

**Применение**:
- Иконка устройства: цвет с прозрачностью **70%**
- Фон иконки: цвет с прозрачностью **8%**
- Вертикальная полоса категории: градиент цвета устройства (70% → 30%)

### 8.5. UX улучшения
- **Дефолтное значение полюсов:** При добавлении нового устройства используется `2P` (фаза + нейтраль) как наиболее распространенный стандарт в РФ.
- **Сортировка групп:** Устройства отображаются в порядке добавления пользователем, без принудительной пересортировки по типу.
- **Иконки в диалогах:** Выпадающее меню выбора типа устройства содержит цветные иконки, соответствующие основному интерфейсу.
