# Руководство по Дизайну (UI/UX Design Guide)

Этот документ описывает визуальные стандарты и принципы взаимодействия (UX) приложения Smart Electric CRM. Все новые компоненты должны соответствовать данным правилам для поддержания «премиального» вида и удобства работы экспертов.

---

## 1. Философия и Эстетика
- **Стиль:** Material 3 с элементами Glassmorphism и «Solid & Clean» дизайна.
- **Ощущение:** Профессиональный инструмент, который выглядит «дорого».
- **Принципы:** Скорость ввода, визуальная иерархия, мгновенная обратная связь.

| Цвет | Использование | Тональность |
| :--- | :--- | :--- |
| **Indigo** | Основной цвет (Primary), заголовки этапов, **весь инженерный раздел (щиты)**, кнопки PDF. | Акцентный, глубокий. |
| **Teal** | Слаботочные щиты (header), Текстовые отчеты. | Спокойный, технологичный. |
| **Green** | Работы, итоговые суммы "Наши". | Положительный, финансовый успех. |
| **Blue** | Материалы, системные уведомления. | Информационный, чистый. |
| **Orange** | Силовые щиты (header), Наценка, критические акценты. | Предупреждающий, энергичный. |
| **Purple** | LED щиты (header). | Современный, ассоциируется с RGB. |
| **Red** | Деструктивные действия (удаление). | Требует осторожности. |
| **Dark Grey** | Контент, вторичные действия (`0xFF374151`). | Нейтральный, фокусирующий. |

### 1.1 Цветовая схема этапов (Stage Colors V3)
Каждый этап имеет цветовую кодировку в списках и диалогах (полоса слева/иконка):

| Этап | Цвет (Material) | Значение |
| :--- | :--- | :--- |
| **Предпросчет** | BlueGrey | Подготовка, черновик. |
| **Этап 1, 2, 1+2** | Blue | Основные черновые работы. |
| **Этап 3** | Green | Чистовой монтаж, успех. |
| **Доп. работы** | Purple | Внеплановые задачи. |
| **Другое** | Amber | Технические или прочие записи. |

## 3. Стандарты Диалоговых Окон (Premium Dialogs)

Все диалоги в приложении должны следовать единой структуре «Premium Dialog».

### 3.1. Анатомия диалога
- **Контейнер (`buildPremiumContainer`):**
    - Скругление углов: **24px**.
    - Максимальная ширина: обычно **400px** (или 720px для предпросмотра).
    - Тень (`BoxShadow`): Цвет темы с прозрачностью **0.15**, размытие (blur) **20px**, смещение по Y **10px**.
- **Шапка (`buildPremiumHeader`):**
    - Фон: Цвет темы с прозрачностью **0.12**.
    - Состав: Иконка (слева), Центрированный заголовок (Bold, 18px), Кнопка закрытия (справа).
    - Скругление верхних углов: **24px**.

### 3.2. Поля ввода
- **Стиль:** Чистые поля без заливки фона.
- **Метки:** Обязательное использование плавающих меток (`floating labels`).
- **Скругление:** Границы полей скруглены на **12px**.

- **Скругление:** Границы полей скруглены на **12px**.

### 3.3. Премиальные Выпадающие Списки (Custom Dropdowns)
Стандартные виджеты `DropdownButton` и `PopupMenuButton` имеют визуальные артефакты (отступы, разделители, серый фон M3). Для обеспечения премиального вида используется паттерн **Custom Trigger + Metadata Menu**.

- **Триггер (Input Field):**
    - Реализация: `InkWell` внутри `Container` с декорацией.
    - Курсор: Строго `SystemMouseCursors.click` (Hand cursor).
    - Визуализация: Иконка стрелки (`arrow_drop_down`) всегда справа.
- **Меню (Popup):**
    - Вызов: Метод `showMenu` с точным позиционированием (`LayoutBuilder`).
    - Стиль:
        - `surfaceTintColor: Colors.transparent` (отключение тонирования M3).
        - `padding: EdgeInsets.zero` (у элементов меню).
        - Элементы меню обернуты в `Container` для отступов.
        - **Фон (Engineering):** Используется акцентный `shade50` (синеватый для Indigo), чтобы меню визуально сливалось с триггером-кнопкой (стиль «Solid Accent»).
    - Контент:
        - Иконки: Цветные иконки для визуальной дифференциации (Flash для тока, Bolt для полюсов).
        - Разделители: `Divider` не используется, границы задаются стилем контейнера.

### 4.1. Вкладки и Списки
- **TabBar:** Использует первичные цвета вкладок (Зеленый для Работ, Синий для Материалов).
- **Карточки позиций:**
    - Группировка по категориям каталога.
    - Компактный вид с четким разделением объемов (Всего / Мы / Контрагент).

### 4.2. Кнопки AppBar и Overflow Menu
- **Прямые действия (Icon Buttons):**
    - **Стиль:** Минимализм, только иконка. Размер 28px.
    - **Иконки:** PDF (Deep Purple), Text (Orange/Teal).
- **Overflow Menu (Три точки):**
    - **Контейнер:** `RoundedRectangleBorder(borderRadius: 16)`, `elevation: 4`, `surfaceTintColor: white`.
    - **Группировка:** Разделители (`PopupMenuDivider`) между логическими блоками (Настройки, Инструменты, Опасное).
    - **Условные элементы:** Пункты (например, "Показывать цены") скрываются, если не применимы к текущей вкладке.
    - **Опасные действия:** Красный цвет иконок и текста, всегда внизу списка.

### 4.3. Хедер Объекта «Об объекте» (Premium Info Header V6)
- **Заголовок:** «Об объекте» (ранее «Объект»).
- **Конструкция:** Белый контейнер с закруглением **20px** и индиго-полосой (6px) слева.
- **Инфо-элементы:** 
    - **2 элемента** (Заказчик, Источник) расположены **вертикально** с цветными иконками в кружках.
    - Адрес и Домофон **удалены** из хедера детального экрана (показываются в карточке списка).
    - **Заказчик:** Использует `SelectableText` для возможности выделения и копирования (телефон, имя).
- **Типографика:** Согласована с карточками этапов (см. п. 4.4).

### 4.3.1. Карточка объекта в списке (Project List Card)
- **Layout:**
    - Адрес: Крупный жирный текст (16px, Bold, Black87), **всегда на первом месте**.
    - Домофон: Мелкий текст (11px, Grey 500) под адресом с префиксом «домофон: ...»
    - **Центрирование:** Если домофон отсутствует, адрес центрируется по вертикали (`Center` widget).
- **Даты:**
    - «Создан: DD.MM.YYYY» — показывается всегда (справа).
    - «Изменен: DD.MM.YYYY» — показывается **только если прошло ≥ 2 часа** после создания.
    - Шрифт: 10px, Grey 400, выравнивание по правому краю.
- **Иконки типов объектов:**
    - Новостройка: `Icons.apartment`
    - Вторичка: `Icons.home`
    - Коттедж: `Icons.villa`
    - Офис: `Icons.business`
    - Другое: `Icons.category` (**отличается** от Icons.business)
- **Hover:** Мягкое затемнение фона, курсор `pointer`.
- **Навигация:** Tap открывает детальный экран объекта.

### 4.3.2. Поиск и Фильтры (AppBar Pattern)
- **Поиск (Adaptive Search):**
    - **Desktop:** Постоянное поле в AppBar.
    - **Mobile:** Раскрывающаяся иконка.
    - **Стиль поля:**
        - `filled: true`, `fillColor`: `surfaceContainerHighest` (c opacity 0.5).
        - `borderRadius`: **24px**.
        - `contentPadding`: Симметричные отступы.
- **Фильтры:**
    - Иконка `filter_list` всегда доступна в AppBar (справа).
    - **Badge:** Маленький индикатор (8px) при активных фильтрах.
    - **Диалог:** `ChoiceChip` с `shrinkWrap` и стилизацией под бренд.

### 4.4. Карточки Этапов (Stage Cards V3)
- **Вертикальные отступы:** Минимальные для высокой плотности данных.
- **Даты (Created/Updated):** Расположены в правой части карточки, выровнены по правому краю.
- **Подписи дат:** "Создан:", "Изменен:" (вместо сокращений).
- **Шрифты:** 
    - Название: 16px Bold, letterSpacing -0.3.
    - Подписи (Labels): 11px, Grey 500, Weight 500.
    - Значения (Values): 14px, Bold/Semi-bold.
- **Hover:** Мягкое затемнение фона.

### 4.5. Диалог добавления этапа (Add Stage Dialog V2)
- **Элементы:** Представлены в виде отдельных карточек ("Premium Cards").
- **Иконка действия:** «+» находится **справа** от названия.
- **Компактность:** Высота элементов уменьшена (padding 10) для исключения скролла при стандартном наборе этапов.

### 4.6. Плавающая кнопка (Floating Action Button)
- **Единая кнопка:** Экран должен содержать **только одну** основную FAB ("Добавить").
- **Вторичные действия:** Переносятся в AppBar (Search, Filters, Menu).
- **Стиль:** Стандартный FAB с тенью `elevation: 4`.

## 6. Интерактивность и Обратная связь
- **Курсор:** Всегда использовать `SystemMouseCursors.click` для кнопок, чипов и интерактивных карточек.
- **Hover-эффект:** Использование прозрачного наложения цвета фона (`fg.withOpacity(0.1)`) при наведении.
- **Подтверждения:** Любое массовое или деструктивное действие (удаление всего, применение шаблона) требует вызова `ConfirmationDialog`.
- **Навигация (Pattern):** После успешного сохранения данных (например, при создании шаблона) диалог должен закрываться, возвращая пользователя на экран, с которого был вызван, без лишних промежуточных окон и без перезагрузки всего дерева (устранен «двойной pop»).

## 8. Инженерный раздел (Shield Management)

### 8.1. Визуальная иерархия щитов
### 8.1. Визуальная иерархия щитов (Shield Card UI)
- **Акцентная полоса (Accent Stripe):** Вертикальная полоса 4px слева, кодирующая тип щита:
    - **Power (Силовой):** Blue (`Color(0xFF3B82F6)`).
    - **LED (Светодиодный):** Purple (`Color(0xFF8B5CF6)`).
    - **Multimedia (Слаботочный):** Amber (`Color(0xFFF59E0B)`).
- **Шапка и Контент (Header & Content):**
    - **Фон:** Нейтральный белый (`Colors.white`) или светло-серый (`Colors.grey.shade50`).
    - **Интерактивность:** При раскрытии (Expand) шапка и граница карточки окрашиваются в легкий оттенок акцентного цвета (Opacity 0.08 / 0.3).
    - **Иконка типа:** Всегда акцентного Indigo цвета, на фоне с прозрачностью 0.08.
- **Типографика (Label-Value Style):**
    - **Label (Верхняя строка):** Тип щита (`POWER`, `LED`). Шрифт `11px`, `Grey 500`, `Weight 500`, `.toUpperCase()`, `letterSpacing: 0.5`.
    - **Value (Нижняя строка):** Название щита. Шрифт `16px`, `Bold`, `Black87`, `letterSpacing: -0.3`. Копирует стиль заголовка «Адрес объекта» для визуального единства.
- **Топ-блок (Top Row):**
    - **Слева (Stats):** Иконка конфигурации (`tune`) серая. Текст статистики нейтральный.
    - **Справа (Mounting):** Переключатель типа монтажа.
- **Зонирование:** Контент щита обернут в контейнер с серым фоном (`Colors.grey.shade50`) и границей для четкого выделения на белом фоне карточки.

### 8.2. Концепция Индиго и Акцентов
- **Унификация диалогов:** Все диалоговые окна инженерного раздела (AddShield, EditShield, ShieldGroup, LedZone) используют **Индиго** как базовый акцентный цвет для кнопок, иконок и заголовков.
- **Фон полей и кнопок:** Вместо прозрачности (`opacity 0.08`) используется сплошной `shade50` (акцентная подложка) для «премиального» и более материального вида.
- **Основные действия (Template, Notes, Delete):**
    - **Шаблоны (`copy_all_rounded`):** Серый outline стиль.
    - **Заметки (`note_alt_outlined`):**
        - *Пустой:* Серый outline.
        - *С заметкой:* Окрашивается в акцентный цвет щита, пульсирует при открытии, имеет индикатор (точку).
- **Кнопка «ДОБАВИТЬ»:**
    - Расположена в **заголовке списка** внутри контента щита.
    - Выровнена по правому краю.
    - Стиль: `OutlinedButton`, акцентный цвет с низкой прозрачностью.

### 8.3. Группировка
- Использование тонких разделителей и отступов (8px) для сохранения компактности.
- Заголовки групп и зон — темно-серый `Color(0xFF374151)` для читаемости на белом фоне.

### 8.4. Цветовое кодирование устройств
Каждый тип устройства в силовых щитах имеет уникальный цвет для мгновенной визуальной идентификации:

| Тип устройства | Цвет | Семантика |
|----------------|------|-----------|
| Рубильники (`load_switch`) | Red.shade600 | Критический элемент, главный выключатель |
| УЗО (`rcd`) | Amber.shade700 | Предупреждение, защита от утечки |
| Автоматы (`circuit_breaker`) | Blue.shade600 | Стандартная защита, стабильность |
| Диф. автоматы (`diff_breaker`) | Purple.shade600 | Премиум защита, комбинированная |
| Реле (`relay`) | Teal.shade600 | Автоматизация, технологии |
| Контакторы (`contactor`) | Orange.shade700 | Силовая коммутация, высокая мощность |
| Другое (`other`) | BlueGrey.shade600 | Нейтральность |

**Применение**:
- Иконка устройства: цвет с прозрачностью **70%**
- Фон иконки: цвет с прозрачностью **8%**
- Вертикальная полоса категории: градиент цвета устройства (70% → 30%)

### 8.5. UX улучшения
- **Дефолтное значение полюсов:** При добавлении нового устройства используется `2P` (фаза + нейтраль) как наиболее распространенный стандарт в РФ.
- **Сортировка групп:** Устройства отображаются в порядке добавления пользователем, без принудительной пересортировки по типу.
- **Иконки в диалогах:** Выпадающее меню выбора типа устройства содержит цветные иконки, соответствующие основному интерфейсу.

## 9. Финансовый монитор (Finance Dashboard)

### 9.1. Общая структура
- **Цветовая схема:** Зеленые акценты (`Color(0xFF2E7D32)`) для элементов финансов.
- **Компактность:** Padding карточек `8px` (вертикальный). При раскрытии спойлера (expand) карточка проекта фиксируется **зеленой рамкой** (`width: 2.0`, `opacity: 1.0`) для визуального обособления.

### 9.2. Плашка количества этапов
- **Дизайн:** Иконка `layers_outlined` + число этапов.
- **Фон:** Градиент от `Color(0xFF2E7D32).withOpacity(0.1)` до `Color(0xFF2E7D32).withOpacity(0.05)`.
- **Обводка:** `Color(0xFF2E7D32).withOpacity(0.3)`, ширина `1px`.
- **Скругление:** `12px`.
- **Размер иконки:** `14px`.
- **Шрифт:** `12px`, font-weight `600`, цвет `Color(0xFF2E7D32)`.

### 9.3.- **Кнопка подтверждения (Оплата):**
  - Тип: `OutlinedButton.icon` для единообразия с кнопками сохранения.
  - Цвет: Ярко-зеленый (`0xFF43A047`).
  - Иконка: `Icons.check_circle` (14px).
  - Эффект: Hover-overlay с прозрачностью **0.08**.
  - Навигация: InkWell с `borderRadius: BorderRadius.circular(8)`.

### 9.4. Типография дат и сумм
- **Итоговая сумма проекта:**
  - Шрифт: `12px`, font-weight `600`.
  - Цвет: **Чисто черный** (`Colors.black`) — для акцента на финальной стоимости.
- **Дата изменения:** 
  - Шрифт: `11px`, font-weight `500`.
  - Цвет: `Colors.grey[600]`.
  - Формат: Динамический ("Сегодня", "Вчера", "3 дн. назад", "DD.MM.YYYY").
- **Сумма контрагента:**
  - Шрифт: `11px`, font-weight `500`.
  - Цвет: `Colors.grey[600]`.
  - Формат: "из 350$" (дополнительная информация под основной суммой).

### 9.5. Адаптивные текстовые поля (Finance)
- **Настройки:** `minLines: 2`, `maxLines: null`.
- **Внутренние отступы (Padding):** `horizontal: 14`, `vertical: 10`.
- **Поведение:** Поля автоматически расширяются по содержимому без фиксированной высоты.
- **Маржи:** Единый стандарт отступов для блоков — **16px**.
- **Применение:** Глобальные поля настроек ("Смета контрагента", "Финансовые заметки").

### 9.6. Выравнивание и Интерактивность
- **Центрирование:** Весь контент внутри строки этапа (текст, суммы, индикаторы, кнопки) строго **центрирован по вертикали**.
- **Зона клика:** Вся строка этапа является интерактивной (`InkWell`), подчеркивая единство данных в строке.
- **Обновление данных:** Pull-to-refresh (`Color(0xFF2E7D32)`). Таймер удален в пользу реактивности.

## 10. Сметы (Estimate Tabs UI)

Блок заметок в сметах синхронизирован по стилистике с панелью «Итого».

### 10.1. Контейнер заметок
- **Фон:** Цвет темы (Зеленый/Синий) с прозрачностью `primaryColorLight.withOpacity(0.5)`.
- **Рамка:** Точно соответствует рамкам блока Итого (`opacity: 0.12`).
- **Скругление:** `10px`.
- **Отступы (Internal):** Нижний отступ увеличен до **20px** для визуального баланса.

### 10.2. Поля ввода
- **Высота:** `minLines: 2`.
- **Типография:** Компактный шрифт `12px`.
- **Цвет поля:** Насыщенный оттенок темы (`_primaryColor.withOpacity(0.08)`), создающий контраст с подложкой.
- **Отступы поля:** `horizontal: 12`, `vertical: 8`.

### 10.3. Кнопки сохранения
- **Тип:** `OutlinedButton.icon` (как в финансах).
- **Логика:** Кнопка появляется только при наличии несохраненных изменений.
- **Спец. логика (Материалы):** Скрытие кнопки, если текст соответствует дефолтному ("Не учтен вводной кабель").

## 11. Статистика (Statistics UI)

### 11.1. График динамики (Work Dynamics Chart)
*   **Layout (Sandwich):**
    *   *Top:* Скрытая легенда (для баланса отступов).
    *   *Center:* График `LineChart`.
    *   *Bottom:* Видимая легенда (USD).
*   **Стиль графика:**
    *   **Линия:** Зеленая (`Colors.green`), сглаженная (`isCurved: true`), толщина 3px.
    *   **Заливка (Gradient):** Градиент от `greenAccent.shade700` (opacity 0.2) к прозрачному.
    *   **Точки (Dots):** Скрыты (show: false) для чистоты, появляются только при наведении (в тултипе).
*   **Тултипы (Tooltips):**
    *   **Фон:** Полупрозрачный черный (`BlueGrey` darker).
    *   **Контент:**
        *   Дата: `dd.MM.yyyy` (белый, bold).
        *   Сумма: `XXXX $` (зеленый акцент).
*   **Ось X:**
    *   Подписи дат (`dd.MM.yyyy` или `MM.yyyy`) серым цветом (10px).
    *   Интервал рассчитывается динамически, чтобы избежать наложения.
*   **Подпись:** "* За выполненные работы" (italic, grey) под графиком.

### 11.2. Круговые диаграммы
*   **Стиль:** `PieChart` с вырезом (Donut chart).
*   **Секции:**
    *   Радиус активной секции увеличивается при нажатии.
    *   Цвета: Набор из Material цветов (Indigo, Orange, Blue, Teal).
*   **Легенда:**
    *   Справа от диаграммы.
    *   Маркеры: Круглые (`BoxShape.circle`).
    *   Текст: Название + Сумма.
