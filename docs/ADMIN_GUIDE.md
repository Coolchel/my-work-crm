# Руководство администратора (Automation Mapping)

Этот документ описывает, как настроить каталог и справочники, чтобы автоматизация смет работала корректно.

## 1. Что влияет на автоматизацию
Ключевые поля `CatalogItem`:
- `mapping_key` — связь инженерных данных с материалами/работами.
- `aggregation_key` — группировка материалов в одну работу.
- `related_work_item` — прямая связь материал -> работа (1-к-1).

## 2. Шаблон ключей для импорта из силового щита

Формат:
`shield_{device_type}_{poles}`

Примеры:
- `shield_circuit_breaker_1P`
- `shield_circuit_breaker_2P`
- `shield_diff_breaker_2P`
- `shield_rcd_2P`
- `shield_rcd_4P`
- `shield_relay_2P`
- `shield_relay_4P`
- `shield_load_switch_1P`
- `shield_other_X`

Рекомендация: используйте единый набор `device_type`, совпадающий со значениями backend (`ShieldGroup.DEVICE_CHOICES`).

## 3. Ключи корпусов щитов

### 3.1 Силовые корпуса
`shield_enclosure_{size}_{mounting}`

Примеры:
- `shield_enclosure_24_internal`
- `shield_enclosure_36_external`

### 3.2 Слаботочные/LED корпуса
`shield_media_enclosure_{size}_{mounting}`

Примеры:
- `shield_media_enclosure_24_internal`
- `shield_media_enclosure_48_external`

Где:
- `size`: стандартный размер (2, 4, 6, 8, 12, 24, 36, 48, 60, 72, 96, 120, 144).
- `mounting`: `internal` или `external`.

## 4. Быстрая настройка нового типа позиции

### Шаг 1. Создайте работу в каталоге
- `item_type = work`
- Задайте цену/единицу.
- Для агрегации при необходимости назначьте `mapping_key` вида `agg_*`.

### Шаг 2. Создайте материал
- `item_type = material`
- Задайте цену/единицу.
- Укажите `mapping_key` (для импорта из инженерки) или `aggregation_key`/`related_work_item` (для расчета работ).

### Шаг 3. Проверьте сценарий
- Импорт материалов из инженерки.
- Расчет работ по материалам.
- Проверка, что дубли не накапливаются, а позиции заменяются корректно.

## 5. Настройка агрегации материалов в работу
Задача: несколько материалов должны формировать одну работу.

Пример:
1. Создайте работу (например, «Прокладка кабеля») и задайте ей `mapping_key = agg_cable_standard`.
2. У материалов-кабелей укажите `aggregation_key = agg_cable_standard`.
3. После запуска расчета система создаст одну work-позицию с суммарным количеством.

## 6. Что делать, если позиция не найдена
Если ключ не сопоставился, в смете создается предупреждающая строка с ценой `0`.

Порядок действий:
1. Посмотрите текст предупреждения и определите недостающий ключ.
2. Добавьте/исправьте `CatalogItem` с нужным `mapping_key`.
3. Повторите импорт/расчет.
4. При необходимости удалите старую предупреждающую строку вручную.

## 7. Directory (Системные словари)

### 7.1 Bootstrap
- Endpoint: `POST /api/directory-sections/bootstrap/`.
- Назначение: синхронизировать choices backend в редактируемые словари.

### 7.2 Поведение при неготовой БД
Если таблицы Directory не созданы:
- list -> пустой массив,
- bootstrap/CRUD -> `503` с читаемым сообщением.

## 8. Работа с кодировкой текста
Если в данных появились искаженные строки (mojibake), выполните:

```bash
python manage.py repair_text_encoding
```

Проверка без сохранения:

```bash
python manage.py repair_text_encoding --dry-run
```

## 9. Контрольный чек-лист после изменений
- `mapping_key` заполнен у позиций, участвующих в импорте.
- `aggregation_key`/`related_work_item` корректно связаны с работами.
- Нормально проходит импорт из щитов.
- Нормально проходит расчет работ.
- В справочниках нет битого текста.
