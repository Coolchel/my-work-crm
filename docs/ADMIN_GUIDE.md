# Руководство администратора системы (Automation Mapping)

Для корректной работы автоматического импорта из инженерной карты (Щиты) в смету, необходимо настроить справочник товаров (`CatalogItem`).
Ключевым полем для связки является `mapping_key`.

## Правила формирования ключей

Система формирует ключ поиска автоматически на основе атрибутов устройства в щите.
Формат ключа: `shield_{device_type}_{poles}`

### Таблица ключей

| Тип устройства (UI) | Device Type (Code) | Полюса (Poles) | Итоговый Mapping Key | Пример товара в каталоге |
| :--- | :--- | :--- | :--- | :--- |
| Автоматический выключатель | `circuit_breaker` | `1P` | `shield_circuit_breaker_1P` | Автомат. выключатель 1P |
| Автоматический выключатель | `circuit_breaker` | `2P` | `shield_circuit_breaker_2P` | Автомат. выключатель 2P |
| Автоматический выключатель | `circuit_breaker` | `3P` | `shield_circuit_breaker_3P` | Автомат. выключатель 3P |
| Диф.автомат | `diff_breaker` | `2P` | `shield_diff_breaker_2P` | Диф.автомат 2P |
| УЗО | `rcd` | `2P` | `shield_rcd_2P` | УЗО 2P |
| УЗО | `rcd` | `4P` | `shield_rcd_4P` | УЗО 4P |
| Реле напряжения | `relay` | `2P` | `shield_relay_2P` | Реле напряжения 1-фазное |
| Реле напряжения | `relay` | `4P` | `shield_relay_4P` | Реле напряжения 3-фазное |
| Выключатель нагрузки | `load_switch` | `1P` | `shield_load_switch_1P` | Рубильник 1P |
| Выключатель нагрузки | `load_switch` | `2P` | `shield_load_switch_2P` | Рубильник 2P |
| Выключатель нагрузки | `load_switch` | `3P` | `shield_load_switch_3P` | Рубильник 3P |
| Выключатель нагрузки | `load_switch` | `4P` | `shield_load_switch_4P` | Рубильник 4P |
| Другое | `other` | `X` | `shield_other_X` | Доп. оборудование |

### Корпуса щитов

Корпуса подбираются автоматически по сумме модулей. Ключ зависит от стандартного размера.

| Размер (Modules) | Mapping Key |
| :--- | :--- |
| 2 | `shield_enclosure_2` |
| 4 | `shield_enclosure_4` |
| 8 | `shield_enclosure_8` |
| 12 | `shield_enclosure_12` |
| 18 | `shield_enclosure_18` |
| 24 | `shield_enclosure_24` |
| 36 | `shield_enclosure_36` |
| ... | и так далее |

## Что делать, если товар не найден?

Если система не находит товар с нужным ключом, она создаст в смете позицию-заглушку с именем:
`ВНИМАНИЕ: Не найден в каталоге! ([Тип] [Полюса] [Номинал])` 
с ценой 0.

**Действия:**
1. Посмотрите в название заглушки, чтобы понять, какой ключ не найден.
2. Создайте в каталоге соответствующий товар.
3. Пропишите ему верный `mapping_key`.
4. Нажмите "Импорт" еще раз — заглушка должна исчезнуть (или обновиться, лучше удалить её вручную перед повторным импортом для чистоты, хотя merge по имени может продублировать, если имя заглушки отличается от имени товара). 
*Примечание: Система попытается найти товар снова. Если найдет, создаст правильную строку. Заглушку лучше удалить вручную.*
