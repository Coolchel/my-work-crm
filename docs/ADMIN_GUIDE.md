# Руководство администратора системы (Automation Mapping)

Для корректной работы автоматического импорта из инженерной карты (Щиты) в смету, необходимо настроить справочник товаров (`CatalogItem`).
Ключевым полем для связки является `mapping_key`.

## Правила формирования ключей

Система формирует ключ поиска автоматически на основе атрибутов устройства в щите.
Формат ключа: `shield_{device_type}_{poles}`

### Таблица ключей

| Тип устройства (UI) | Device Type (Code) | Полюса (Poles) | Итоговый Mapping Key | Пример товара в каталоге |
| :--- | :--- | :--- | :--- | :--- |
| Автоматический выключатель | `circuit_breaker` | `1P` | `shield_circuit_breaker_1P` | Автомат. выключатель 1P |
| Автоматический выключатель | `circuit_breaker` | `2P` | `shield_circuit_breaker_2P` | Автомат. выключатель 2P |
| Автоматический выключатель | `circuit_breaker` | `3P` | `shield_circuit_breaker_3P` | Автомат. выключатель 3P |
| Диф.автомат | `diff_breaker` | `2P` | `shield_diff_breaker_2P` | Диф.автомат 2P |
| УЗО | `rcd` | `2P` | `shield_rcd_2P` | УЗО 2P |
| УЗО | `rcd` | `4P` | `shield_rcd_4P` | УЗО 4P |
| Реле напряжения | `relay` | `2P` | `shield_relay_2P` | Реле напряжения 1-фазное |

### Корпуса щитов (Enclosures)

Система подбирает корпус на основе:
1. **Типа щита:** Силовой (`shield_enclosure_`) или Слаботочный/LED (`shield_media_enclosure_`).
2. **Размера:** Ближайший стандартный размер (24, 36, ... 144).
3. **Монтажа:** `internal` (Встраиваемый) или `external` (Навесной).

#### Матрица ключей (Примеры)

| Тип | Монтаж | Размер | Mapping Key | Имя в смете (Пример) |
| :--- | :--- | :--- | :--- | :--- |
| **Силовой** | Встройка | 24 | `shield_enclosure_24_internal` | Щит силовой на 24 модуля (встройка) |
| **Силовой** | Навесной | 36 | `shield_enclosure_36_external` | Щит силовой на 36 модулей (наружный) |
| **Media** | Встройка | 24 | `shield_media_enclosure_24_internal` | Щит слаботочный на 24 модуля (встройка) |
| **Media** | Навесной | 48 | `shield_media_enclosure_48_external` | Щит слаботочный на 48 модулей (наружный) |

*(И так далее для всех размеров: 2, 4, 6, 8, 12, 18, 24, 36, 48, 60, 72, 96, 120, 144)*

## Что делать, если товар не найден?

Если система не находит товар с нужным ключом, она создаст в смете позицию-заглушку с именем:
`ВНИМАНИЕ: Не найден корпус в каталоге! ({SIZE} мод, {MOUNTING})` 
или
`ВНИМАНИЕ: Не найден слаботочный корпус! ({SIZE} мод, {MOUNTING})`
с ценой 0.

**Действия:**
1. Посмотрите в название заглушки, чтобы понять, какой ключ не найден.
2. Создайте в каталоге соответствующий товар.
3. Пропишите ему верный `mapping_key`.
4. Нажмите "Импорт" еще раз — заглушка должна исчезнуть (или обновиться, лучше удалить её вручную перед повторным импортом для чистоты, хотя merge по имени может продублировать, если имя заглушки отличается от имени товара). 
*Примечание: Система попытается найти товар снова. Если найдет, создаст правильную строку. Заглушку лучше удалить вручную.*

## Агрегация Материалов в Работы
Для автоматического создания работ из материалов используется два механизма:
1. **Агрегация (Группировка):** Объединение нескольких материалов под одной работой.
2. **Прямая связь (Legacy):** 1-к-1 связь через поле "Связанная работа".

### Настройка Агрегации
Чтобы настроить группировку (например, все кабели -> одна работа "Прокладка"):

1. **Создайте Работу** в каталоге (например, "Прокладка кабеля").
   - Задайте ей `mapping_key` = `agg_cable_standard` (любой уникальный ключ с префиксом `agg_`).
2. **Настройте Материалы:**
   - Для каждого кабеля (3x1.5, 3x2.5) в поле `aggregation_key` пропишите тот же ключ: `agg_cable_standard`.

**Результат:**
При расчете сметы, система создаст **одну** строчку работы, сложив количества всех материалов с этим ключом.

| Поле в Admin | Назначение |
| :--- | :--- |
| **Технический ключ (`mapping_key`)** | У **Работы**: идентификатор, по которому она будет найдена (например, `agg_cable`). |
| **Ключ агрегации (`aggregation_key`)** | У **Материалов**: ссылка на ключ работы. Все материалы с одинаковым ключом будут просуммированы. |
