# План рефакторинга раздела "Справочник"

## Входные артефакты
- Изучены документы: `docs/PRD.md`, `docs/LOGIC.md`, `docs/DESIGN.md`.
- Файл `spec.md` в репозитории не найден (проверка выполнена по имени в корне и в `docs/`).

## Цели
1. Полный CRUD для справочника на backend.
2. Добавить инфраструктуру БД для системных справочников (типы, статусы, категории и т.д.).
3. Пересобрать frontend-экран справочника в стиле текущих разделов (компактные строки, премиальные диалоги).
4. Дать пользователю возможность редактировать данные базы вручную из приложения.

## Детальный пошаговый план
1. Проанализировать существующие backend-модели и выделить все текущие "разделы" из `choices`.
2. Добавить в backend новые модели для универсального справочника разделов и позиций.
3. Добавить REST CRUD для новых моделей и endpoint синхронизации (`bootstrap`) для автозаполнения системных разделов.
4. Проверить совместимость существующего API каталога (категории/позиции) и расширить CRUD-операции.
5. Переработать frontend раздела справочника:
   - вкладка системных разделов + CRUD позиций,
   - вкладка каталога + CRUD категорий и элементов.
6. Сделать компактные плитки и диалоги в стиле текущего приложения.
7. Выполнить самопроверку, зафиксировать отчет и подготовить PR.
